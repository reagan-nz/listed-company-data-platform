# 路线图

## 第一阶段：年报数据获取与基础字段抽取（当前）

**目标**：从巨潮资讯网稳定获取 A 股年报 PDF，抽取 11 项基础字段，建立证据链。

| 事项 | 状态 |
|---|---|
| CNINFO 年报查询与 PDF 下载 | 已完成 |
| 11 字段确定性抽取器 | 已完成 |
| 4 公司泛化验证 + 通用修复 | 已完成 |
| 200 家分层评估 | 已完成 |
| 1000 家受控评估 + 严格审计 | 已完成 |
| rnd / 表格 / 金融 schema 优化 | 进行中 |

**完成标准**：非金融 strict-usable ≥ 93%，rnd ≥ 80%，可批量跑全 A 股。

## 第二阶段：全 A 股基础数据库

**目标**：覆盖 5500+ 家上市公司，产出统一结构的 `company_profile.json` 数据库。

- 批量跑全 A 股（复用 eval1000 缓存与断点续跑机制）
- 统一入库格式（SQLite / Parquet / JSONL，待定）
- 按板块/行业/年份索引
- 失败公司清单与重试策略（no_announcement、no_text_layer、network error）
- 定期增量更新（新年报披露季）

**完成标准**：≥ 90% A 股有 ≥ 9/11 字段 strict-usable。

## 第三阶段：字段质量提升与行业 schema 拆分

**目标**：针对不同行业披露差异，提升字段准确率与覆盖率。

- 金融公司独立 schema（银行/券商/保险/期货）
- rnd_investment 数值抽取规则收紧
- 表格解析增强（合并单元格、header-only 检测、preview 切片）
- concentration 聚合比优先
- 北交所 / 科创板特殊模板适配
- 持续人工校准（每 milestone 60–100 格）

**完成标准**：全 A 股 strict-usable ≥ 95%，各字段 strict ≥ 85%。

## 第四阶段：BrowserUser 爬虫智能体补充数据

**目标**：用 BrowserUser 获取 PDF 抽取无法覆盖的数据，扩充数据库宽度与深度。

- 投资者互动平台（e 互动 / 上证 e 互动）
- 公司官网 IR 页面（非 PDF 披露）
- 交易所公告（非年报类）
- 政府采购 / 招投标
- 专利商标（国知局公开数据）
- 新闻与舆情（标题 + 链接，不存全文）

**原则**：BrowserUser 仅补充，不替代已稳定的 PDF 抽取；强反爬/付费/需登录源先记录为限制。

**完成标准**：至少 2 个新数据源试点成功，产出可入库的结构化记录。

## 第五阶段：公司分析、变化追踪与产业链线索

**目标**：基于基础数据库，提供分析能力。

- 跨年度字段变化追踪（主营业务迁移、R&D 趋势、客户集中度变化）
- 行业聚合统计（板块 R&D 中位数、区域收入分布）
- 产业链线索挖掘（客户/供应商名称 → 关联公司）
- 简单查询 API 或导出接口
- 可视化 dashboard（可选）

**完成标准**：支持按行业/板块的聚合查询与单公司时间序列对比。
