# v0.4 1000 家受控评估方案

_状态：已完成（2026-06-18）_

## 背景

200 家评估验证了概念并驱动了关键修复，但样本量不足以确认修复在大规模下的稳定性。需要 ~1000 家受控评估，且必须是 200 家的 strict superset 以保持可比性。

## 目标

1. ~1000 家分层评估，验证改进后 pipeline 稳定性
2. 200 家子集可直接对比（0 回归）
3. 金融公司单独标记与统计
4. 全量 strict 二次审计

## 方案

### 样本设计

```bash
# 5× 分层抽样（同 seed → 200 是 1000 的子集）
python lab/sample_universe.py --scale 5 --seed 20260617 \
  --out lab/eval_companies_1000_raw.yaml

# union with eval200 → 1020 家（200 strict superset + 820 新增）
# 金融公司标记：is_financial(name) → financial: true
```

| 板块 | 200 家 | 1000 家（×5） |
|---|---|---|
| sse_main | 60 | 300 |
| star | 25 | 125 |
| szse_main | 50 | 250 |
| chinext | 45 | 225 |
| bse | 20 | 100 |

### 缓存复用

- 从 eval200 复制 184 份 PDF + meta.json + .cache 到 eval1000
- 仅 ~836 家需新下载
- 预计耗时 ~2–3 小时（1 req/s throttle）

### 评估执行

```bash
python lab/eval_generalize.py \
  --companies lab/eval_companies_1000.yaml \
  --out outputs/generalization/eval1000 \
  --throttle 1.0
```

- 后台运行，VPN 断开可重跑（断点续跑）
- 1 家 network timeout（601901）后续重试成功

### 校准与审计

- 60 格人工校准（seed=1000）
- 全量 9937 plausible 单元格 strict adversarial 审计

## 结果

| 指标 | 数值 |
|---|---|
| 样本 | 1020 家 |
| 成功 | 946 家 |
| no_announcement | 73 家 |
| hard error | 0 |
| 非金融 proxy plausible | 10.5/11 (96%) |
| strict-usable | **10.16/11 (92.4%)** |
| plausible 中实质正确 | 96.4% |
| hard-wrong | 1.9% |
| 200 子集 | 27 改进，0 回归 |

### 各字段 strict 率（非金融，936 家）

| 字段 | proxy | strict |
|---|---|---|
| rnd_investment | 79.3% | **67.7%** |
| revenue_by_region | 96.0% | **90.7%** |
| revenue_by_segment | 98.3% | 95.7% |
| 其余 8 字段 | ≥90% | ≥90% |

## 结论

- **方向可行**：92.4% strict-usable 证明年报程序化抽取可以支撑基础数据库
- **proxy 高估约 3–4pp**：不能直接用 plausible 当准确率
- **最弱环节明确**：rnd_investment（67.7%）和 revenue 表格（90.7%）
- **可以进入第二阶段**（全 A 股），但建议先收紧 rnd 和表格规则

## 遗留

- rnd_investment 数值规则需收紧（TOTAL 标签 + 实质金额）
- 表格 preview 需 data-row 校验
- 金融 schema 未拆分
- eval_results.json（1.9MB）不适合提交 Git
