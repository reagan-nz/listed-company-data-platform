# v0.3 200 家评估方案

_状态：已完成（2026-06-17）_

## 背景

4 公司测试验证了概念可行性，但样本太小无法发现系统性问题。需要在中等规模（~200 家）上验证泛化能力，并建立可复现的评估与校准流程。

## 目标

1. 200 家分层抽样评估，测量 proxy plausible 率
2. 建立人工校准流程，验证 proxy 可信度
3. 发现并修复最高优先级问题

## 方案

### 抽样设计

```bash
python lab/sample_universe.py --seed 20260617 \
  --out lab/eval_companies_200.yaml
```

| 板块 | 目标数 |
|---|---|
| sse_main（沪主板） | 60 |
| star（科创板） | 25 |
| szse_main（深主板） | 50 |
| chinext（创业板） | 45 |
| bse（北交所） | 20 |
| **合计** | **200** |

- 从 CNINFO 股票列表 API 获取 orgId（跳过后续 resolve 步骤）
- seed=20260617 保证可复现

### 评估执行

```bash
python lab/eval_generalize.py \
  --companies lab/eval_companies_200.yaml \
  --out outputs/generalization/eval200 \
  --throttle 1.0
```

- 184/200 成功（16 家 no_announcement 导致 hard crash）
- 断点续跑：已缓存 PDF 跳过网络

### 人工校准

```bash
python lab/calibration_sample.py \
  --eval-dir outputs/generalization/eval200 --n 40 --seed 42
```

- 40 格分层样本（60% plausible / 40% not）
- 人工 PDF 对照评分
- 引入 MISSED 分级（extractor 返回 not_found 但字段实际存在）

## 结果

### 发现的问题（按优先级）

| 优先级 | 问题 | 影响 |
|---|---|---|
| P0 | hard crash（16 家 no_announcement TypeError） | 8% 崩溃率 |
| P1 | major_products plausible 仅 59% | 最大单字段缺口 |
| P2 | A+H 报告选错（中国银行 1/11） | 双上市公司 |
| P3 | risk_factors 召回不足 | 91% → 需扩展锚点 |
| P4 | revenue_by_region 误匹配 | 境外资产/合并表 |

### 修复（均在 eval200 后完成）

1. **no_announcement guard**：`_retry` 返回 None 时不再 crash
2. **major_products fallback_anchors**：从业务概述 sibling section 回退
3. **A+H pick_full_report**：优先 A 股，降级 H 股
4. **risk_factors 锚点扩展** + pointer avoid
5. **revenue_by_region**：移除 bare 境内/境外 anchor + combined-table preview 切片

### 校准结果（v3 graded）

- proxy precision（CORRECT only）：93%
- false-positive rate：4%
- calibrated population correctness：88%

## 遗留

- 200 样本仍偏小，需 1000 家验证修复是否 generalize → 触发 v0.4
- rnd_investment 问题在 200 家中已显现（79%）但未深入修复
- eval200 的 eval_results.json 基于修复前 pipeline，数字已 stale
