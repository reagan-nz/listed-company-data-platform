# CNINFO P0 小样本公司选择方案

> **Sub Issue 1：准备 CNINFO P0 验证样本公司清单。** 本文件定义 P0 小样本验证阶段的公司样本选择规则，服务后续最新公告列表、公告 PDF 元数据、个股 F10 / 公司资料三项验证任务。

## 1. 目的

本文件用于定义 CNINFO P0 小样本验证阶段的公司样本选择规则。目标是确保后续最新公告列表、公告 PDF 元数据、个股 F10 / 公司资料验证覆盖不同板块、行业和公司类型，而不是随机选择少数公司。

需要强调：

- 本文件只定义**样本选择方案**；
- **不代表**已经完成真实抓取；
- **不代表**样本公司数据已经验证；
- **不做**数据库接入；
- **不写**爬虫。

## 2. 样本规模

建议样本规模：

- **第一批基础样本**：30–50 家公司；
- 如果公告 PDF 样本不足，可从公告结果中额外扩展 PDF 样本；
- F10 / 公司资料验证优先复用同一批公司；
- 最新公告验证与 PDF 验证尽量使用相同公司，方便关联。

样本量**不追求覆盖全市场**，而是为了发现字段可得性、页面结构差异、失败原因和访问风险。

## 3. 样本覆盖维度

| 覆盖维度 | 需要覆盖的类型 | 为什么需要覆盖 |
|---|---|---|
| 交易所 / 板块 | 上交所主板、深交所主板、创业板、科创板、北交所 | 不同板块页面结构、orgId 映射、公告入口可能不同 |
| 行业 | 制造业、信息技术、医药、消费、金融或类金融、能源 / 材料；可选：环保 / 公用事业 | 行业分类口径、F10 字段展示、公告类型分布可能不同 |
| 公司状态 | 正常上市公司；ST / *ST / 风险类公司；可能存在公告异常或年报缺失风险的公司 | 风险类公司页面标识、公告类型、字段完整性可能不同 |
| 公告频率 | 公告频率较高公司；公告频率较低公司 | 高频公司可测试分页与列表稳定性；低频公司可测试空结果与边界情况 |
| 上市时间 | 上市时间较长公司；近年上市公司 | 老公司与新公司的 F10 资料完整度、历史公告回溯可能不同 |
| 公司规模 | 大市值公司；中小市值公司 | 大公司资料更完整，小公司可能字段缺失或页面简化 |

## 4. 样本公司记录字段

| 字段 | 说明 | 是否必填 |
|---|---|---|
| `company_code` | 股票代码 | 是 |
| `company_name` | 公司名称或证券简称 | 是 |
| `exchange` | 交易所，例如 SSE / SZSE / BSE | 是 |
| `board` | 板块，例如主板 / 创业板 / 科创板 / 北交所 | 是 |
| `industry` | 行业 | 是 |
| `listing_status` | 上市状态 | 是 |
| `is_st` | 是否 ST / *ST | 是 |
| `market_cap_group` | 大 / 中 / 小市值分组；如暂时没有真实市值数据，可先用 `unknown` | 否 |
| `announcement_frequency_group` | 公告频率分组，高 / 中 / 低；如暂时没有统计可先用 `unknown` | 否 |
| `sample_reason` | 为什么选择这家公司 | 是 |
| `used_for_latest_announcement_validation` | 是否用于最新公告列表验证 | 是 |
| `used_for_pdf_metadata_validation` | 是否用于公告 PDF 元数据验证 | 是 |
| `used_for_f10_validation` | 是否用于 F10 / 公司资料验证 | 是 |
| `notes` | 补充说明 | 否 |

## 6. 后续如何生成真实样本

后续可以从已有全市场公司列表或 CNINFO 年报抽取结果中选择真实公司。

建议步骤：

1. 从已有公司 universe 中读取公司代码和名称；
2. 按交易所 / 板块分组；
3. 按行业分组；
4. 人工加入少量 ST / 风险类公司；
5. 人工检查是否覆盖北交所、科创板、创业板；
6. 输出 30–50 家样本清单；
7. 后续验证任务统一复用该清单。

如果部分字段暂时没有，例如市值或公告频率，可以先标记 `unknown`，**不影响第一轮验证**。

## 7. 与后续验证任务的关系

样本清单完成后，后续任务按顺序执行：

1. 使用样本公司验证 CNINFO 最新公告列表（参见 [cninfo_data_source_value_inventory.md](./cninfo_data_source_value_inventory.md) 第 8.5 节）；
2. 从最新公告结果中抽取公告 PDF 样本（参见第 8.6 节）；
3. 使用同一批公司验证 F10 / 公司资料（参见第 8.7 节）；
4. 最后汇总字段可得性、失败原因和 `recommended_status`；
5. 更新 `docs/data_sources.md`。
