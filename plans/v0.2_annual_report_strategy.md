# v0.2 年报抽取策略（Plan B）

_状态：已完成（2026-06-16）_

## 背景

v0.1 多源验证发现 CNINFO 年报 PDF 是最有价值的单源。v0.2 将项目重心从「多源覆盖率验证」转向「年报 PDF 基础字段确定性抽取」。

## 目标

从 A 股年报 PDF 中抽取 11 项基础字段，每条数据可证据追溯到 PDF 具体页码和原文。

## 方案

### 核心设计

1. **field_schema.py**：11 字段定义，每项含 anchors（锚点）、region（MD&A/附注）、extraction 类型、avoid 规则
2. **extract_annual_report.py**：确定性抽取器
   - PyMuPDF 文本层 + pdfplumber 表格层
   - `locate_section()` 锚点评分定位
   - 4 种抽取：section_snippet / table / numeric / concentration
3. **证据链**：每个字段输出 value + evidence_sentence + page + source_url

### 11 字段

mda, industry_discussion, main_business_segments, major_products, revenue_by_segment, revenue_by_region, top_customers, top_suppliers, rnd_investment, major_subsidiaries, risk_factors

### 4 公司泛化测试

| 公司 | 代码 | 行业 | 目的 |
|---|---|---|---|
| 宁德时代 | 300750 | 新能源 | 调参参考 |
| 三一重工 | 600031 | 机械 | 不同工业 |
| 招商银行 | 600036 | 银行 | 最难 case |
| 澜起科技 | 688008 | 半导体 | 科创板 |

### 5 项通用修复（从 4 公司测试中发现）

1. **Prose-first concentration**：客户/供应商从叙述句提取（「前五名…占…%」），不要求表格
2. **avoid 负上下文**：锚点后跟「在职员工」等 → 降分，避免误匹配
3. **Narrative anchors 优先**：main_business_segments 用「报告期内公司从事的主要业务」而非「主营业务」
4. **Header-aware table selection**：table_match tokens 区分 region vs segment 表
5. **Full-report filter**：probe 跳过摘要/取消/更正，选全文年报

## 结果

| 公司 | 修复前 | 修复后 |
|---|---|---|
| CATL | 11/11 | 11/11（无回归） |
| 三一重工 | 5/11 | **10/11** |
| 招商银行 | 3/11 + 2 absent | **4/11 + 2 absent** |
| 澜起科技 | 8/11 | **9/11** |

工业/科技公司 out-of-sample 准确率约 9–10/11；银行 gracefully degrade。

## 遗留

- 金融 schema 未拆分
- rnd_investment 数值规则粗糙
- 未做大规模评估（仅 4 家） → 触发 v0.3
